Introduction
What is Async Programming?
Before writing a single line of async code, you need to understand why it exists and what problem it solves.

The Problem: Waiting Does Nothing

Almost every real program spends time waiting: waiting for a file to load from disk, for a database to return results, for an HTTP server to respond. During that wait, the CPU is idle — it could be doing useful work, but instead it just sits there.

This is called I/O-bound waiting. It's the core problem that async programming solves.

💡 Two Kinds of Bottlenecks

I/O-bound — Your program waits for network, disk, or database. The CPU is mostly idle. This is where async shines.

CPU-bound — Your program does heavy computation: image processing, ML, cryptography. The CPU runs at full speed. Async does not help here — you need multiprocessing.

What Does Async Actually Do?

Async programming lets a single thread interleave multiple operations. Instead of blocking (doing nothing) while waiting for I/O, the program says "while I'm waiting for this, go do something else — and come back when it's ready."

Start request A
→
Waiting for A
(yields control)
→
Start request B
→
Waiting for B
→
A done → resume
→
B done → resume

Both operations overlap in time. If each takes 1 second, they finish in ~1 second total instead of 2.

Python's Approach: asyncio

Python implements async programming through the asyncio module (built into the standard library since Python 3.4, and matured significantly in 3.7+). The syntax uses two keywords:

async def

Declares a coroutine function — a function that can pause and resume. Calling it returns a coroutine object; it doesn't run yet.

await

Inside an async def, suspends the current coroutine and yields control until the awaited thing completes. This is the pause point.

Sync vs Async
Chapter 1
Synchronous vs Asynchronous
See the concrete difference in code — and in timing — between blocking and non-blocking programs.

Synchronous Code: One Thing at a Time

In synchronous (normal) Python, each line runs to completion before the next one starts. If line 3 blocks for 2 seconds, everything after it waits.

Python — synchronous 
import time

def fetch_user(user_id):
    print(f"Fetching user {user_id}...")
    time.sleep(2)          # simulates a 2-second network call
    return f"User:{user_id}"

def main():
    start = time.time()

    u1 = fetch_user(1)   # blocks for 2 seconds
    u2 = fetch_user(2)   # then blocks for another 2 seconds
    u3 = fetch_user(3)   # then another 2 seconds

    print(f"Got: {u1}, {u2}, {u3}")
    print(f"Total time: {time.time() - start:.1f}s")

main()
Output
Fetching user 1...
Fetching user 2...
Fetching user 3...
Got: User:1, User:2, User:3
Total time: 6.0s

Six seconds — even though each fetch is independent and could have run at the same time. That's wasted time.

Asynchronous Code: Interleaved Waiting

Now the same program rewritten with async/await. The three fetches overlap:

Python — asynchronous 
import asyncio, time

async def fetch_user(user_id):
    print(f"Fetching user {user_id}...")
    await asyncio.sleep(2)   # yields control; event loop runs others
    print(f"Done: user {user_id}")
    return f"User:{user_id}"

async def main():
    start = time.time()

    # All three start immediately; waiting overlaps
    u1, u2, u3 = await asyncio.gather(
        fetch_user(1),
        fetch_user(2),
        fetch_user(3),
    )

    print(f"Got: {u1}, {u2}, {u3}")
    print(f"Total time: {time.time() - start:.1f}s")

asyncio.run(main())
Output
Fetching user 1...
Fetching user 2...
Fetching user 3...
Done: user 1
Done: user 2
Done: user 3
Got: User:1, User:2, User:3
Total time: 2.0s

Three fetches in 2 seconds instead of 6. The waiting time overlaps because all three are suspended at their await points simultaneously, and the event loop resumes each one when its wait is done.

⚠ Key Distinction: Concurrency ≠ Parallelism

The code above runs on one thread. It's not running three things at once in the CPU sense. It's interleaving waits — while one coroutine waits for I/O, another runs. True parallelism (multiple CPU cores) requires the multiprocessing module.

The Event Loop
Chapter 2
The Event Loop
The engine that makes async Python work. Understanding it demystifies everything else.

What Is the Event Loop?

The event loop is a scheduler that runs on a single thread. It maintains a queue of coroutines and tasks that are ready to run, executes them one at a time, and watches for I/O completion events.

Its job, repeated endlessly:

Pick a ready coroutine from the queue
If nothing is ready, wait until an I/O event fires.
Run it until the next await
Coroutines run synchronously up to their first suspension point.
Register what it's waiting for
The OS kernel monitors the socket/file descriptor for completion.
Move on to the next ready coroutine
Other work proceeds while the previous coroutine waits.
When I/O finishes, re-queue the waiting coroutine
It resumes from exactly where it paused, with the result.

Visualizing the Loop

Python — event loop in action 
import asyncio

async def task_a():
    print("A: start")
    await asyncio.sleep(1)   # A suspends; loop runs B
    print("A: resumed after 1s")

async def task_b():
    print("B: start")
    await asyncio.sleep(0.5)  # B suspends; loop checks A
    print("B: resumed after 0.5s")

async def main():
    await asyncio.gather(task_a(), task_b())

asyncio.run(main())
Output — notice B resumes before A
A: start
B: start
B: resumed after 0.5s
A: resumed after 1s

A and B both start. B's wait is shorter, so it resumes first. A resumes later. All on one thread, perfectly ordered by the event loop.

One Loop Per Thread

Python's asyncio uses one event loop per thread. You almost always run just one, created by asyncio.run(). You can introspect it:

Python 
import asyncio

async def check():
    loop = asyncio.get_running_loop()
    print(f"Loop type: {type(loop).__name__}")
    print(f"Is running: {loop.is_running()}")

asyncio.run(check())
# Loop type: _UnixSelectorEventLoop   (on Linux/Mac)
# Loop type: ProactorEventLoop         (on Windows)
# Is running: True
⚠ Never block the event loop

The entire async model collapses if you run a blocking operation (like time.sleep() or requests.get()) inside a coroutine. It freezes the loop completely — no other coroutines can run. Always use async-aware alternatives: await asyncio.sleep(), aiohttp, etc. More on this in Chapter 13.

Coroutines & async def
Chapter 3
Coroutines & async def
Coroutines are the foundation of Python's async model. Here's exactly what they are and how they behave.

What Is a Coroutine?

A coroutine is a function that can pause its execution midway, yield control to other code, and later resume from exactly where it paused — retaining all its local variables.

You create a coroutine function by prefixing def with async:

Python 
async def greet(name: str) -> str:
    return f"Hello, {name}!"

# Calling it does NOT execute the body — returns a coroutine object
coro = greet("Alice")
print(coro)         # <coroutine object greet at 0x...>
print(type(coro))   # <class 'coroutine'>

# To actually run it:
result = asyncio.run(coro)
print(result)       # Hello, Alice!
⚠ Common Beginner Mistake

Calling greet("Alice") looks like calling a normal function, but it returns a coroutine object — the body hasn't run at all. Python will even warn you: RuntimeWarning: coroutine 'greet' was never awaited if you forget to run it.

Coroutines Can Call Other Coroutines

Use await inside a coroutine to call another coroutine. This is how you compose async code:

Python 
import asyncio

async def get_user_id(username: str) -> int:
    await asyncio.sleep(0.3)   # simulates DB lookup
    return 42

async def get_profile(user_id: int) -> dict:
    await asyncio.sleep(0.3)   # simulates another DB lookup
    return {"id": user_id, "name": "Alice", "role": "admin"}

async def load_user(username: str) -> dict:
    user_id = await get_user_id(username)   # await another coroutine
    profile = await get_profile(user_id)    # await another coroutine
    return profile

async def main():
    user = await load_user("alice")
    print(user)   # {'id': 42, 'name': 'Alice', 'role': 'admin'}

asyncio.run(main())

Type Annotations with Coroutines

Use Coroutine or simply annotate the return type of the value it produces — Python infers the coroutine wrapper:

Python
from collections.abc import Coroutine

# These two are equivalent; the second is simpler and preferred
async def fetch() -> str: ...       # preferred
async def fetch() -> Coroutine[None, None, str]: ...  # verbose, rarely needed

The await Keyword
Chapter 4
The await Keyword
await is the moment a coroutine pauses. Understanding exactly what it does — and what it can be used on — is essential.

What await Does

When Python executes await expr, three things happen:

  1. The current coroutine suspends — its local state is frozen in memory.
  2. Control returns to the event loop, which can run other coroutines.
  3. When the awaited operation finishes, the event loop resumes this coroutine with the result.
Python — await mechanics 
import asyncio

async def compute():
    x = 10
    print(f"x is {x}, about to suspend")

    # Suspends here. x = 10 is preserved in memory.
    await asyncio.sleep(1)

    # Resumes here. x is still 10.
    print(f"Resumed. x is still {x}")
    return x * 2

result = asyncio.run(compute())
print(f"Result: {result}")   # 20

What Can You await?

You can await any awaitable object. There are three kinds:

Awaitable typeExampleDescription
Coroutineawait fetch(url)Result of calling an async def function
asyncio.Taskawait taskA coroutine scheduled to run concurrently
asyncio.Futureawait futureLow-level placeholder for an eventual result

Any object that implements __await__ is also awaitable (used by library authors, rarely by application developers).

await Only Works Inside async def

❌ SyntaxError
def sync_function():
    # await in a normal function
    result = await something()
    # SyntaxError!
✅ Correct
async def async_function():
    # await inside async def
    result = await something()
    return result  # works

Chained Awaits — Sequential Execution

Multiple awaits in a row run one after another. Use this when each step depends on the previous result:

Python — sequential awaits 
async def process_order(order_id: int):
    # Step 1: validate (must complete before step 2)
    valid = await validate_order(order_id)
    if not valid:
        raise ValueError("Invalid order")

    # Step 2: charge (uses result of step 1)
    payment_id = await charge_customer(order_id)

    # Step 3: dispatch (uses payment_id)
    tracking = await dispatch(order_id, payment_id)

    return tracking

Even though each step suspends at its await, from the perspective of this coroutine they appear sequential — exactly like reading normal synchronous code. That's the beauty of async/await syntax.

What happens when Python reaches await asyncio.sleep(2)?

✅ Exactly right! The coroutine suspends, freeing the event loop to run anything else that's ready. After 2 seconds, the event loop resumes this coroutine.

asyncio.run()
Chapter 5
asyncio.run()
Every async program needs a single entry point. asyncio.run() is it.

What asyncio.run() Does

asyncio.run(coro) does four things in order:

Creates a new event loop
Allocates the event loop infrastructure for this thread.
Runs the coroutine to completion
Drives the event loop until coro returns or raises.
Cancels any remaining tasks
Cleans up background tasks that weren't explicitly awaited.
Closes and destroys the loop
Frees resources. Returns whatever the coroutine returned.
Python — complete program structure 
import asyncio

async def setup():
    await asyncio.sleep(0.1)
    print("Setup complete")

async def teardown():
    await asyncio.sleep(0.1)
    print("Teardown complete")

async def main():          # ← top-level async entry point
    await setup()
    print("Running application...")
    await teardown()

# ── Program entry point ──────────────────────────────
if __name__ == "__main__":
    asyncio.run(main())    # creates loop, runs main(), closes loop

Rules for asyncio.run()

✅ Best Practices
  • Call it exactly once in your program, at the top level.
  • Always pass it a coroutine object — not an already-running one.
  • Don't nest it: never call asyncio.run() inside an async def. Use await there.
❌ Wrong — nested run()
async def main():
    # Don't do this!
    asyncio.run(helper())
    # RuntimeError: This event loop
    # is already running
✅ Correct
async def main():
    # Use await inside async
    await helper()
    # ✓ correct

Return Values

asyncio.run() returns whatever the coroutine returned:

Python 
async def compute() -> int:
    await asyncio.sleep(0)
    return 42

result = asyncio.run(compute())
print(result)   # 42

async with & async for
Chapter 6
async with & async for
Two async-aware versions of Python's context managers and iterators — essential for working with resources like HTTP sessions and streams.

async with — Async Context Managers

The regular with statement calls __enter__ and __exit__ on a context manager. async with calls the async versions: __aenter__ and __aexit__ — which are coroutines themselves and can await things (like closing a connection).

Python — async with 
import asyncio
import aiohttp

async def fetch(url: str) -> str:
    # aiohttp.ClientSession is an async context manager
    async with aiohttp.ClientSession() as session:
        # session.get() also returns an async context manager
        async with session.get(url) as response:
            return await response.text()
    # both resources are properly closed/released on exit

Writing Your Own Async Context Manager

Python 
class AsyncDBConnection:
    async def __aenter__(self):
        print("Opening DB connection...")
        await asyncio.sleep(0.1)   # simulate async connect
        return self                  # value bound to 'as' variable

    async def __aexit__(self, exc_type, exc, tb):
        print("Closing DB connection...")
        await asyncio.sleep(0.1)   # simulate async close

    async def query(self, sql: str):
        await asyncio.sleep(0.05)
        return ["row1", "row2"]

async def main():
    async with AsyncDBConnection() as db:
        rows = await db.query("SELECT * FROM users")
        print(rows)
    # connection is closed even if an exception is raised

asyncio.run(main())

async for — Async Iterators

async for iterates over objects that produce values asynchronously — like reading lines from a network stream or processing a queue of events. The iterator's __anext__ method is a coroutine.

Python — async for with a stream 
async def line_stream(lines):
    """Async generator that yields lines with a simulated delay."""
    for line in lines:
        await asyncio.sleep(0.1)   # simulate network read delay
        yield line

async def main():
    data = ["first line", "second line", "third line"]

    async for line in line_stream(data):
        print(f"Received: {line}")

asyncio.run(main())
💡 Async Generators

A function with both async def and yield is an async generator. It's automatically an async iterator — no need to implement __aiter__ and __anext__ manually. Perfect for streaming data.

gather()
Chapter 7
asyncio.gather() — Run in Parallel
When multiple coroutines are independent of each other, run them concurrently with gather().

The Problem gather() Solves

Sequential awaits run one after another — even when there's no dependency between them. gather() starts all of them at once and waits until all finish:

Sequential — slow
async def main():
  a = await fetch("A")  # 1s
  b = await fetch("B")  # 1s
  c = await fetch("C")  # 1s
  # Total: 3 seconds
gather() — fast
async def main():
  a, b, c = await asyncio.gather(
      fetch("A"),  # 1s ─┐
      fetch("B"),  # 1s  ├ overlap
      fetch("C"),  # 1s ─┘
  )
  # Total: ~1 second

gather() Guarantees

  • Returns a list of results in the same order as the input coroutines — regardless of which finished first.
  • If any coroutine raises an exception, it propagates immediately (by default) and cancels the rest.
  • You can unpack results directly: a, b, c = await asyncio.gather(f1(), f2(), f3())
Python — complete gather() example 
import asyncio, time

async def fetch_price(ticker: str, delay: float) -> dict:
    print(f"  Requesting {ticker}...")
    await asyncio.sleep(delay)
    print(f"  Got {ticker}")
    return {"ticker": ticker, "price": round(100 + delay * 10, 2)}

async def main():
    start = time.time()

    results = await asyncio.gather(
        fetch_price("AAPL", 0.8),
        fetch_price("GOOG", 1.2),
        fetch_price("MSFT", 0.5),
    )

    print(f"\nAll done in {time.time()-start:.2f}s")
    for r in results:
        print(f"  {r['ticker']}: ${r['price']}")
    # Results are in input order (AAPL, GOOG, MSFT)
    # even though MSFT finished first

asyncio.run(main())

Handling Partial Failures with return_exceptions=True

Python 
async def might_fail(n: int):
    await asyncio.sleep(0.1)
    if n == 2: raise ValueError("Something went wrong")
    return f"ok-{n}"

async def main():
    results = await asyncio.gather(
        might_fail(1),
        might_fail(2),   # this one fails
        might_fail(3),
        return_exceptions=True   # don't propagate — collect as value
    )
    for r in results:
        if isinstance(r, Exception):
            print(f"Error: {r}")
        else:
            print(f"Success: {r}")

# Output:
# Success: ok-1
# Error: Something went wrong
# Success: ok-3

Chapter 8
Tasks & create_task()
Tasks schedule a coroutine to run immediately in the background, without waiting for it upfront.

Coroutine vs Task

A coroutine doesn't start until you await it. A Task starts running immediately when created — it's scheduled on the event loop right away, and you can await it later to get the result.

Python 
import asyncio

async def background_sync():
    print("Sync: started")
    await asyncio.sleep(1)
    print("Sync: complete")
    return "sync done"

async def main():
    # Create a Task — it starts running NOW
    task = asyncio.create_task(background_sync())

    # We can do other things while it runs
    print("Main: doing other work")
    await asyncio.sleep(0.5)
    print("Main: still doing things")

    # Now wait for the task to finish and get its result
    result = await task
    print(f"Main: task returned '{result}'")

asyncio.run(main())
Output
Sync: started
Main: doing other work
Main: still doing things
Sync: complete
Main: task returned 'sync done'

Task Naming (Python 3.7+)

Name your tasks for easier debugging:

Python
task = asyncio.create_task(sync_data(), name="data-sync")
print(task.get_name())   # "data-sync"

Inspecting and Cancelling Tasks

Python — task lifecycle methods 
task = asyncio.create_task(long_operation())

task.done()       # True if finished or cancelled
task.cancelled()  # True if cancelled
task.result()    # Get result (raises if not done or exception)
task.exception() # Get the exception (if any)
task.cancel()    # Request cancellation

TaskGroup Python 3.11+

asyncio.TaskGroup is the modern, safer way to run multiple tasks. If any task raises, all others are cancelled automatically:

Python 3.11+ 
async def main():
    async with asyncio.TaskGroup() as tg:
        task1 = tg.create_task(fetch("A"))
        task2 = tg.create_task(fetch("B"))
        task3 = tg.create_task(fetch("C"))
    # All three complete (or all are cancelled) before here
    print(task1.result(), task2.result(), task3.result())

Timeouts & Cancellation
Chapter 9
Timeouts & Cancellation
Real programs can't wait forever. Cancellation is also the mechanism behind timeouts, task groups, and graceful shutdown.

Timeouts with asyncio.wait_for()

Python 
import asyncio

async def slow_operation():
    await asyncio.sleep(10)    # takes 10 seconds
    return "finally done"

async def main():
    try:
        result = await asyncio.wait_for(
            slow_operation(),
            timeout=3.0           # give it 3 seconds
        )
        print(result)
    except asyncio.TimeoutError:
        print("Timed out after 3 seconds")

asyncio.run(main())

asyncio.timeout() Context Manager Python 3.11+

Cleaner syntax for applying a deadline to a whole block of code:

Python 3.11+ 
async def main():
    try:
        async with asyncio.timeout(5.0):   # 5-second deadline
            a = await step_one()
            b = await step_two(a)
            print(b)
    except TimeoutError:
        print("The whole block exceeded 5 seconds")

How Cancellation Works

When a task is cancelled, Python injects a CancelledError at the next await point inside that coroutine. It propagates up unless you catch it:

Python — cancellation and cleanup 
import asyncio

async def long_running():
    try:
        print("Starting long work...")
        await asyncio.sleep(100)   # CancelledError injected here
        print("This never prints")
    except asyncio.CancelledError:
        print("Cleaning up resources...")
        await asyncio.sleep(0)   # you can await during cleanup
        print("Cleanup done")
        raise                      # always re-raise CancelledError!

async def main():
    task = asyncio.create_task(long_running())
    await asyncio.sleep(1)
    task.cancel()
    try:
        await task
    except asyncio.CancelledError:
        print("Task was cancelled")

asyncio.run(main())
⚠ Always re-raise CancelledError

Swallowing CancelledError (catching it without re-raising) breaks cancellation propagation. The task will appear to be cancelled from the outside but actually continues running. Always raise after your cleanup code.

Queues & Synchronization
Chapter 10
Queues & Synchronization
Coordinate multiple coroutines safely with async-aware queues, locks, semaphores, and events.

asyncio.Queue — Producer / Consumer

asyncio.Queue is a thread-safe async queue for passing data between coroutines. Producers put items in; consumers get items out.

Python — producer/consumer pattern 
import asyncio

async def producer(queue: asyncio.Queue, items: list):
    for item in items:
        await asyncio.sleep(0.3)          # simulate work before producing
        await queue.put(item)
        print(f"Produced: {item}")
    await queue.put(None)                  # sentinel value to signal done

async def consumer(queue: asyncio.Queue):
    while True:
        item = await queue.get()            # blocks until item available
        if item is None:
            print("Consumer: done")
            break
        await asyncio.sleep(0.1)          # process the item
        print(f"Consumed: {item}")
        queue.task_done()

async def main():
    queue = asyncio.Queue(maxsize=3)      # buffer up to 3 items
    items = ["alpha", "beta", "gamma", "delta"]
    await asyncio.gather(
        producer(queue, items),
        consumer(queue),
    )

asyncio.run(main())

asyncio.Lock — Mutual Exclusion

Protects a shared resource so only one coroutine accesses it at a time:

Python 
lock = asyncio.Lock()
counter = 0

async def increment():
    global counter
    async with lock:           # only one coroutine at a time
        val = counter
        await asyncio.sleep(0)   # yield, but lock prevents others entering
        counter = val + 1

asyncio.Semaphore — Limit Concurrency

Allows at most N coroutines to run a section simultaneously. Perfect for rate-limiting API calls:

Python — limit to 5 concurrent requests 
sem = asyncio.Semaphore(5)   # max 5 at a time

async def limited_fetch(url: str) -> str:
    async with sem:             # blocks if 5 are already running
        return await do_fetch(url)

async def main():
    urls = [f"https://api.example.com/item/{i}" for i in range(100)]
    results = await asyncio.gather(*[limited_fetch(u) for u in urls])

asyncio.Event — Signal Between Coroutines

Python 
event = asyncio.Event()

async def waiter():
    print("Waiting for signal...")
    await event.wait()        # suspends until event is set
    print("Got the signal!")

async def setter():
    await asyncio.sleep(1)
    event.set()               # all waiters are resumed

Async HTTP with aiohttp
The most common real-world use of async Python — making non-blocking HTTP requests.
Install
pip install aiohttp

Basic GET Request

Python 
import asyncio, aiohttp

async def fetch(url: str) -> dict:
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as resp:
            resp.raise_for_status()     # raise on 4xx/5xx
            return await resp.json()   # parse JSON body

async def main():
    data = await fetch("https://api.github.com/repos/python/cpython")
    print(f"Stars: {data['stargazers_count']}")

asyncio.run(main())

Fetching Many URLs Concurrently

The key: create one session and reuse it across all requests. Creating a new session per request is wasteful.

Python — concurrent fetches with one session 
import asyncio, aiohttp, time

async def fetch_one(session: aiohttp.ClientSession, url: str) -> str:
    async with session.get(url) as resp:
        return await resp.text()

async def fetch_all(urls: list[str]) -> list[str]:
    async with aiohttp.ClientSession() as session:
        # Pass the SAME session to all coroutines
        tasks = [fetch_one(session, url) for url in urls]
        return await asyncio.gather(*tasks)

async def main():
    urls = [
        "https://httpbin.org/delay/1",
        "https://httpbin.org/delay/1",
        "https://httpbin.org/delay/1",
    ]
    start = time.time()
    results = await fetch_all(urls)
    print(f"Fetched {len(results)} URLs in {time.time()-start:.2f}s")
    # ~1 second instead of 3

asyncio.run(main())

async def main():

    pool = await asyncpg.create_pool(
        dsn="postgresql://user:password@localhost/mydb",

Blocking Code
Chapter 13
Running Blocking Code
Sometimes you must use a synchronous library. Here's how to do it without freezing the event loop.

The Problem

Any blocking call inside a coroutine freezes the entire event loop for its duration. No other coroutine can run:

❌ Freezes the event loop
async def bad():
    # Blocks the ENTIRE event loop for 3 seconds!
    time.sleep(3)                  # ❌ blocking sleep
    data = requests.get(url).text  # ❌ blocking HTTP
    text = open("file.txt").read() # ❌ blocking file read

Solution: run_in_executor()

loop.run_in_executor() offloads a synchronous function to a thread pool, and wraps the result as an awaitable. The event loop remains free to run other coroutines while the thread does the blocking work.

Python — run_in_executor 
import asyncio, time, requests
from concurrent.futures import ThreadPoolExecutor

def blocking_fetch(url: str) -> str:
    """This is a regular synchronous function."""
    return requests.get(url).text

async def main():
    loop = asyncio.get_running_loop()

    # None = use the default ThreadPoolExecutor
    result = await loop.run_in_executor(
        None, blocking_fetch, "https://example.com"
    )
    print(result[:100])

asyncio.run(main())

With a Custom Thread Pool

Python 
async def main():
    loop = asyncio.get_running_loop()

    with ThreadPoolExecutor(max_workers=10) as pool:
        results = await asyncio.gather(*[
            loop.run_in_executor(pool, blocking_fetch, url)
            for url in urls
        ])

CPU-Bound Work: ProcessPoolExecutor

For CPU-heavy work (not just I/O), use ProcessPoolExecutor to bypass Python's GIL and run on multiple cores:

Python — CPU-bound offloading 
from concurrent.futures import ProcessPoolExecutor

def cpu_heavy(n: int) -> int:
    """Runs in a separate process — true parallelism."""
    return sum(i * i for i in range(n))

async def main():
    loop = asyncio.get_running_loop()
    with ProcessPoolExecutor() as pool:
        result = await loop.run_in_executor(pool, cpu_heavy, 10_000_000)
    print(result)

asyncio.run(main())
💡 asyncio.to_thread() — Python 3.9+

A cleaner shorthand for run_in_executor(None, ...):

result = await asyncio.to_thread(blocking_function, arg1, arg2)

Common Pitfalls
Chapter 14
Common Pitfalls
The mistakes almost every async Python developer makes at least once — and how to avoid them.

1. Blocking the Event Loop

🚫 Most Common Mistake

Using any blocking call inside a coroutine stalls all other coroutines until it returns.

❌ Wrong
async def handler():
    time.sleep(2)       # ❌
    requests.get(url)  # ❌
    open("f.txt").read() # ❌
✅ Correct
async def handler():
    await asyncio.sleep(2)     # ✅
    await session.get(url)     # ✅
    await f.read()             # ✅ (aiofiles)

2. Forgetting await

❌ Silent bug
async def main():
    result = fetch()   # forgot await!
    print(result)       # prints coroutine object
    # RuntimeWarning: never awaited
✅ Correct
async def main():
    result = await fetch()
    print(result)   # the actual data

3. Sequential Awaits When You Need Concurrency

❌ Still slow (3 seconds)
async def main():
    a = await fetch("A")
    b = await fetch("B")
    c = await fetch("C")
    # B waits for A to finish
✅ Fast (1 second)
async def main():
    a, b, c = await asyncio.gather(
        fetch("A"),
        fetch("B"),
        fetch("C"),
    )

4. Swallowing CancelledError

❌ Breaks cancellation
async def worker():
    try:
        await do_work()
    except Exception:
        pass  # catches CancelledError too!
✅ Correct
async def worker():
    try:
        await do_work()
    except asyncio.CancelledError:
        cleanup()
        raise  # always re-raise!

5. Nested asyncio.run()

❌ RuntimeError
async def outer():
    # Don't call run() inside async!
    asyncio.run(inner())  # ❌
✅ Correct
async def outer():
    await inner()         # ✅

asyncio.run(outer())   # once, at top level

6. Creating a Session Per Request

❌ Wasteful
async def fetch(url):
    # New session every call
    async with aiohttp.ClientSession() as s:
        async with s.get(url) as r:
            return await r.text()
✅ Share one session
async def fetch(session, url):
    # Reuse the same session
    async with session.get(url) as r:
        return await r.text()

The Mental Checklist

✅ Before you ship async code
  • Every async def call is awaited (or wrapped in a Task)
  • No time.sleep(), requests.get(), or blocking file I/O inside coroutines
