Why Python for PDFs?

Extracting Text from a PDF

The most common task: reading what's inside a PDF. We'll use pypdf for simple extraction and pdfplumber for more accurate layout-aware extraction.

Method 1 — pypdf (simple)

python 
from pypdf import PdfReader

# Open the PDF file
reader = PdfReader("my_document.pdf")

# How many pages does it have?
print(f"Total pages: {len(reader.pages)}")

# Extract text from a specific page (page index starts at 0)
first_page = reader.pages[0]
print(first_page.extract_text())

# Extract text from ALL pages
full_text = ""
for page in reader.pages:
    full_text += page.extract_text()

print(full_text)

Method 2 — pdfplumber (more accurate)

pdfplumber is better at preserving the visual layout of text on the page:

python 
import pdfplumber

# Use a "with" block — it automatically closes the file when done
with pdfplumber.open("my_document.pdf") as pdf:

    # Loop through each page
    for i, page in enumerate(pdf.pages):
        print(f"--- Page {i+1} ---")
        text = page.extract_text()
        if text:
            print(text)
        else:
            print("(No text found on this page)")
ℹ️ When to use which

Use pypdf for quick text grabs. Use pdfplumber when spacing and layout accuracy matter, or when you need to extract tables.

Reading PDF Metadata

PDFs carry hidden metadata: author, title, creation date, and more. Here's how to read it:

python 
from pypdf import PdfReader

reader = PdfReader("my_document.pdf")
meta = reader.metadata

print(f"Title:   {meta.title}")
print(f"Author:  {meta.author}")
print(f"Subject: {meta.subject}")
print(f"Creator: {meta.creator}")

# Check if the PDF is encrypted
print(f"Encrypted: {reader.is_encrypted}")
print(f"Pages:     {len(reader.pages)}")

Extracting Tables

pdfplumber can detect and extract table data as Python lists, which you can then convert to a spreadsheet:

python 
import pdfplumber

with pdfplumber.open("report_with_tables.pdf") as pdf:
    for i, page in enumerate(pdf.pages):
        tables = page.extract_tables()

        for j, table in enumerate(tables):
            print(f"Table {j+1} on page {i+1}:")
            for row in table:
                print(row)  # Each row is a Python list

Save Tables to Excel

Install pandas first: pip install pandas openpyxl

python 
import pdfplumber
import pandas as pd

all_tables = []

with pdfplumber.open("report.pdf") as pdf:
    for page in pdf.pages:
        for table in page.extract_tables():
            if table:
                # First row is the header, rest are data rows
                df = pd.DataFrame(table[1:], columns=table[0])
                all_tables.append(df)

if all_tables:
    combined = pd.concat(all_tables, ignore_index=True)
    combined.to_excel("extracted_tables.xlsx", index=False)
    print("Saved to extracted_tables.xlsx ✅")

Merging & Splitting PDFs

Merge multiple PDFs into one

python 
from pypdf import PdfWriter, PdfReader

# List of PDFs to merge
pdf_files = ["chapter1.pdf", "chapter2.pdf", "chapter3.pdf"]

writer = PdfWriter()

for filename in pdf_files:
    reader = PdfReader(filename)
    for page in reader.pages:
        writer.add_page(page)

# Save the merged result
with open("merged_book.pdf", "wb") as output:
    writer.write(output)

print("PDFs merged! ✅")

Split a PDF into individual pages

python 
from pypdf import PdfReader, PdfWriter

reader = PdfReader("big_document.pdf")

# Save each page as a separate file
for i, page in enumerate(reader.pages):
    writer = PdfWriter()
    writer.add_page(page)

    filename = f"page_{i+1}.pdf"
    with open(filename, "wb") as out:
        writer.write(out)
    print(f"Saved {filename}")

print("Split complete! ✅")

Rotating Pages

Use .rotate(degrees) on a page. Valid values: 90, 180, 270.

python 
from pypdf import PdfReader, PdfWriter

reader = PdfReader("sideways.pdf")
writer = PdfWriter()

for page in reader.pages:
    page.rotate(90)  # Rotate 90° clockwise
    writer.add_page(page)

with open("fixed.pdf", "wb") as out:
    writer.write(out)

print("Pages rotated! ✅")

Adding a Watermark

First create a watermark PDF (a transparent "CONFIDENTIAL" stamp, for example), then merge it onto every page:

python 
from pypdf import PdfReader, PdfWriter

# Load the watermark (a one-page PDF with just your stamp)
watermark_reader = PdfReader("watermark.pdf")
watermark_page = watermark_reader.pages[0]

# Apply the watermark to every page of the main document
reader = PdfReader("original.pdf")
writer = PdfWriter()

for page in reader.pages:
    page.merge_page(watermark_page)  # Overlay the watermark
    writer.add_page(page)

with open("watermarked.pdf", "wb") as out:
    writer.write(out)

print("Watermark applied! ✅")
💡 Tip

Create your watermark.pdf using reportlab with a semi-transparent diagonal text — see the "Creating PDFs" section below.

Encrypting & Decrypting PDFs

Add a password

python 
from pypdf import PdfReader, PdfWriter

reader = PdfReader("sensitive.pdf")
writer = PdfWriter()

for page in reader.pages:
    writer.add_page(page)

# Encrypt with a user password and owner password
writer.encrypt(user_password="open123", owner_password="admin456")

with open("protected.pdf", "wb") as out:
    writer.write(out)

print("Password protected! 🔒")

Remove a password

python 
from pypdf import PdfReader, PdfWriter

reader = PdfReader("protected.pdf")

# Provide the password to unlock
reader.decrypt("open123")

writer = PdfWriter()
for page in reader.pages:
    writer.add_page(page)

with open("unlocked.pdf", "wb") as out:
    writer.write(out)

print("Password removed! 🔓")

Creating a Basic PDF

Use reportlab to generate PDFs from scratch. The canvas module gives you full control over text position, fonts, and drawing.

python 
from reportlab.lib.pagesizes import letter
from reportlab.pdfgen import canvas

# Create the canvas — think of it as a blank sheet of paper
c = canvas.Canvas("hello.pdf", pagesize=letter)
width, height = letter  # 612 x 792 points (1 point = 1/72 inch)

# Set font: (font name, size)
c.setFont("Helvetica-Bold", 24)

# Draw text at (x, y) — origin (0,0) is bottom-left corner!
c.drawString(72, height - 100, "Hello, PDF World!")

c.setFont("Helvetica", 12)
c.drawString(72, height - 130, "Created with Python and reportlab.")

# Draw a horizontal line
c.line(72, height - 150, width - 72, height - 150)

# Save the PDF
c.save()
print("PDF created! ✅")
⚠️ Coordinate System

In reportlab, (0, 0) is the bottom-left corner, not the top-left. So to place text near the top of an A4 or letter page, use a Y value close to the page height and subtract.

Creating a Report with Layout

For multi-page documents with automatic text wrapping and layout, use reportlab's Platypus (Page Layout and Typography Using Scripts):

python 
from reportlab.lib.pagesizes import letter
from reportlab.platypus import SimpleDocTemplate, Paragraph, Spacer, PageBreak
from reportlab.lib.styles import getSampleStyleSheet
from reportlab.lib.units import inch

# Set up the document
doc = SimpleDocTemplate("report.pdf", pagesize=letter)
styles = getSampleStyleSheet()
story = []  # A list of content "blocks" to render in order

# Title
story.append(Paragraph("Monthly Sales Report", styles['Title']))
story.append(Spacer(1, 0.3 * inch))  # Vertical spacing

# Heading
story.append(Paragraph("Executive Summary", styles['Heading1']))

# Body paragraph — text wraps automatically!
body_text = """
    This report summarizes the performance of our sales team for the month
    of June 2026. Overall revenue exceeded targets by 12%, driven primarily
    by strong performance in the enterprise segment.
"""
story.append(Paragraph(body_text, styles['Normal']))
story.append(Spacer(1, 0.2 * inch))

# Start a new page
story.append(PageBreak())
story.append(Paragraph("Detailed Breakdown", styles['Heading1']))
story.append(Paragraph("Page 2 content goes here...", styles['Normal']))

# Build the final PDF
doc.build(story)
print("Report created! ✅")

Adding Images & Tables to a PDF

python 
from reportlab.platypus import SimpleDocTemplate, Image, Table, TableStyle, Paragraph
from reportlab.lib.pagesizes import letter
from reportlab.lib import colors
from reportlab.lib.styles import getSampleStyleSheet

doc = SimpleDocTemplate("with_image_table.pdf", pagesize=letter)
styles = getSampleStyleSheet()
story = []

# Add an image (width and height in points)
img = Image("logo.png", width=200, height=100)
story.append(img)

# Add a table — data is a list of rows, each row is a list of cells
data = [
    ["Name",    "Department",  "Sales"],   # Header row
    ["Alice",   "Enterprise",  "$42,000"],
    ["Bob",     "SMB",         "$28,500"],
    ["Charlie", "Partnerships","$35,200"],
]

table = Table(data)
table.setStyle(TableStyle([
    ('BACKGROUND', (0,0), (-1,0), colors.HexColor("#4f46e5")),  # Header bg
    ('TEXTCOLOR',  (0,0), (-1,0), colors.white),                 # Header text
    ('FONTNAME',   (0,0), (-1,0), 'Helvetica-Bold'),
    ('ROWBACKGROUNDS', (0,1), (-1,-1), [colors.white, colors.HexColor("#f3f4f6")]),
    ('GRID', (0,0), (-1,-1), 0.5, colors.HexColor("#d1d5db")),
    ('PADDING', (0,0), (-1,-1), 8),
]))

story.append(table)
doc.build(story)
print("PDF with image and table created! ✅")

OCR: Reading Scanned PDFs

Scanned PDFs are just images — there's no actual text layer. To extract text from them, you need OCR (Optical Character Recognition).

Installation

bash 
# Python packages
pip install pytesseract pdf2image

# Also install Tesseract OCR engine (system package):
# Ubuntu/Debian:
sudo apt install tesseract-ocr poppler-utils

# macOS (Homebrew):
brew install tesseract poppler

Extracting Text from a Scanned PDF

python 
import pytesseract
from pdf2image import convert_from_path

# Step 1: Convert each PDF page to an image
images = convert_from_path("scanned_invoice.pdf", dpi=300)

# Step 2: Run OCR on each image
full_text = ""
for i, image in enumerate(images):
    print(f"Processing page {i+1}...")
    text = pytesseract.image_to_string(image)
    full_text += f"\n--- Page {i+1} ---\n" + text

print(full_text)

# Optionally save to a text file
with open("extracted.txt", "w") as f:
    f.write(full_text)
print("OCR complete! ✅")
⚠️ Note

OCR accuracy depends heavily on scan quality. A DPI of 300 or higher gives the best results. Low-contrast or tilted pages will reduce accuracy.
