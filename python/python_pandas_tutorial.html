pandas is Python's most powerful tool for working with structured data — think of it as a supercharged spreadsheet you control with code.

pip install pandas— then you're ready

What is pandas?

pandas is a Python library built for data manipulation and analysis. It lets you load, clean, filter, transform, and analyze tabular data (rows and columns — like a spreadsheet or database table) efficiently using Python.

📋 Series

A single column of data with an index. Like a Python list, but more powerful.

📊 DataFrame

A table with rows and columns. This is the core object you'll use most.

🏷️ Index

Labels for each row — like a primary key. Can be numbers, dates, or strings.

🔄 Vectorized ops

Operations apply to entire columns at once — no loops needed, very fast.

Series — A Single Column

A Series is a one-dimensional array with labels (an index). Think of it as a single column from a spreadsheet.

python
import pandas as pd

# Create a Series from a Python list
scores = pd.Series([88, 92, 75, 95, 67])
print(scores)
Output
0 88
1 92
2 75
3 95
4 67
dtype: int64
python
# Give it a custom index (labels)
scores = pd.Series(
    [88, 92, 75, 95, 67],
    index=["Alice", "Bob", "Carol", "Dave", "Eve"]
)

print(scores["Alice"])   # 88  — access by label
print(scores.mean())     # 83.4
print(scores.max())      # 95
print(scores > 80)       # Boolean mask

DataFrame — Your Main Tool

A DataFrame is a 2D table — rows and named columns. You can create one from a dictionary, a list of lists, a CSV, or many other sources.

Creating a DataFrame

python
import pandas as pd

# From a dictionary — keys become column names
data = {
    "name":    ["Alice", "Bob", "Carol", "Dave"],
    "dept":    ["Sales", "Tech", "Sales", "HR"],
    "salary":  [72000, 95000, 68000, 61000],
    "years":   [3, 7, 2, 5],
}

df = pd.DataFrame(data)
print(df)
namedeptsalaryyears
0AliceSales720003
1BobTech950007
2CarolSales680002
3DaveHR610005
ℹ️ Index

The leftmost column (0, 1, 2, 3) is the index — row labels that pandas adds automatically. You can customize it or set a column as the index.

Reading CSV & Excel Files

In real projects, you load data from files — not type it by hand. pandas makes this one line of code.

Read a CSV file

python
# Basic — reads header row automatically
df = pd.read_csv("employees.csv")

# With options:
df = pd.read_csv(
    "sales_data.csv",
    sep=",",           # delimiter (default is comma)
    header=0,          # row number of the header (0 = first row)
    index_col="id",    # set a column as the index
    parse_dates=["date"] # auto-parse date columns
)

Read an Excel file

python
# Read the first sheet
df = pd.read_excel("report.xlsx")

# Read a specific sheet by name
df = pd.read_excel("report.xlsx", sheet_name="Q2 Data")

# Read all sheets into a dictionary
sheets = pd.read_excel("report.xlsx", sheet_name=None)
print(sheets.keys())  # dict_keys(['Sheet1', 'Q2 Data', ...])

Inspecting a DataFrame

Before doing anything with your data, always explore it first. These are the commands you'll use every single time:

python
df.head()         # First 5 rows (pass a number to change: df.head(10))
df.tail(3)        # Last 3 rows
df.shape          # (rows, columns) — e.g. (1500, 12)
df.columns        # Column names
df.dtypes         # Data type of each column
df.info()         # Summary: columns, non-null counts, dtypes
df.describe()     # Stats: mean, std, min, max for numeric columns
df.isnull().sum() # Count missing values per column
df.value_counts() # Count occurrences (on a Series or column)
💡 Habit to build

Always run df.head(), df.info(), and df.isnull().sum() as your first three commands whenever you load a new dataset.

Selecting Columns & Rows

Selecting Columns

python
# Single column → returns a Series
df["name"]

# Multiple columns → returns a DataFrame
df[["name", "salary"]]

# All columns except one
df.drop(columns=["years"])

Selecting Rows by Position

python
# Row at position 0 (first row)
df.iloc[0]

# Rows 0 to 2 (not including 3)
df.iloc[0:3]

# Last row
df.iloc[-1]

# Specific row AND columns by position
df.iloc[1, 2]       # row 1, column 2
df.iloc[:3, [0,2]]   # first 3 rows, columns 0 and 2

Filtering Rows

Filtering uses boolean conditions — you write a condition that produces True/False for each row, then use it to select matching rows.

python
# Single condition: salary over 70,000
df[df["salary"] > 70000]

# Multiple conditions: use & (and), | (or)
# ⚠️ Wrap each condition in parentheses!
df[(df["dept"] == "Sales") & (df["salary"] > 65000)]

# Filter with OR
df[(df["dept"] == "Sales") | (df["dept"] == "HR")]

# Shorthand for OR — isin()
df[df["dept"].isin(["Sales", "HR"])]

# NOT equal
df[df["dept"] != "Tech"]

# Filter by string content
df[df["name"].str.startswith("A")]

loc vs iloc

These are the two main ways to select specific rows and columns. The difference: loc uses labels, iloc uses integer positions.

python
# ── loc: select by LABEL ──────────────────────────────
df.loc[0]                  # Row with index label 0
df.loc[0, "salary"]        # Row 0, column "salary"
df.loc[0:2, ["name", "dept"]] # Rows 0–2, two columns
                             # NOTE: loc slicing INCLUDES the end (2)

# ── iloc: select by POSITION ──────────────────────────
df.iloc[0]                  # First row (position 0)
df.iloc[0, 2]              # First row, third column
df.iloc[0:3, 0:2]          # Rows 0–2 (excl 3), cols 0–1 (excl 2)

# ── Update a value ────────────────────────────────────
df.loc[0, "salary"] = 75000  # Change Alice's salary
⚠️ Key difference

loc[0:2] returns rows with labels 0, 1, and 2 (inclusive). iloc[0:2] returns only positions 0 and 1 (exclusive end) — just like regular Python slicing.

Handling Missing Values

Real-world data always has gaps. pandas represents missing values as NaN (Not a Number). Here's how to find and fix them:

python
# ── Detect missing values ─────────────────────────────
df.isnull()          # True wherever value is NaN
df.isnull().sum()    # Count NaNs per column
df.notnull()         # Inverse — True where NOT missing

# ── Drop rows with missing values ─────────────────────
df.dropna()                    # Drop any row with at least one NaN
df.dropna(subset=["salary"])  # Only drop if "salary" is missing
df.dropna(thresh=3)           # Keep rows with at least 3 non-NaN values

# ── Fill missing values ───────────────────────────────
df.fillna(0)                   # Fill all NaNs with 0
df["salary"].fillna(df["salary"].mean())  # Fill with column mean
df.fillna(method="ffill")      # Forward fill (copy previous row's value)
df.fillna(method="bfill")      # Backward fill

# ── In-place vs new DataFrame ─────────────────────────
df_clean = df.dropna()         # Returns a new DataFrame
df.dropna(inplace=True)       # Modifies df directly

Removing Duplicates

python
# Check for duplicate rows
df.duplicated()              # True for every duplicate row
df.duplicated().sum()        # Count of duplicate rows

# Remove duplicates
df.drop_duplicates()                   # Drop fully duplicate rows
df.drop_duplicates(subset=["email"])  # Drop rows where email is repeated
df.drop_duplicates(keep="last")      # Keep last occurrence (default: "first")

Data Types

pandas assigns a data type to each column. Wrong types cause errors and slow performance. Common types: int64, float64, object (string), bool, datetime64.

python
# Check types
df.dtypes

# Convert types
df["salary"] = df["salary"].astype(float)
df["years"]  = df["years"].astype(int)
df["active"] = df["active"].astype(bool)

# Convert a column that might have errors (use errors="coerce")
# Anything that can't convert becomes NaN
df["revenue"] = pd.to_numeric(df["revenue"], errors="coerce")

# Convert a string column to datetime
df["hire_date"] = pd.to_datetime(df["hire_date"])

Rename & Drop Columns

python
# Rename specific columns
df = df.rename(columns={"name": "full_name", "dept": "department"})

# Rename all columns at once (must match count)
df.columns = ["full_name", "department", "salary", "years"]

# Lowercase all column names (very handy!)
df.columns = df.columns.str.lower()

# Drop columns
df = df.drop(columns=["years"])

# Drop rows by index
df = df.drop(index=[0, 3])

# Reset the index after dropping rows
df = df.reset_index(drop=True)

GroupBy — Aggregate by Category

groupby() is one of pandas' most powerful features. It splits the data by a category, applies a function to each group, and combines the results.

python
# Average salary per department
df.groupby("dept")["salary"].mean()

# Multiple stats at once
df.groupby("dept")["salary"].agg(["mean", "min", "max", "count"])

# Different stat per column
df.groupby("dept").agg({
    "salary": "mean",
    "years":  "sum",
    "name":   "count"
})

# Group by multiple columns
df.groupby(["dept", "location"])["salary"].mean()

# Size of each group
df.groupby("dept").size()

# Reset index so dept becomes a regular column again
result = df.groupby("dept")["salary"].mean().reset_index()

Apply Custom Functions

apply() lets you run any function — built-in or your own — on rows or columns.

python
# Apply a built-in function to a column
df["name_upper"] = df["name"].apply(str.upper)

# Apply a lambda (short inline function)
df["salary_k"] = df["salary"].apply(lambda x: x / 1000)

# Apply your own custom function
def salary_band(salary):
    if salary >= 90000:
        return "Senior"
    elif salary >= 70000:
        return "Mid"
    else:
        return "Junior"

df["band"] = df["salary"].apply(salary_band)

# Apply across the whole row (axis=1)
df["summary"] = df.apply(
    lambda row: f"{row['name']} in {row['dept']}",
    axis=1
)
💡 Performance tip

For simple math operations, avoid apply() and use vectorized operations instead — they're 10–100x faster: df["salary"] * 1.1 is much faster than df["salary"].apply(lambda x: x * 1.1).

Merge & Join DataFrames

Combine two DataFrames — just like SQL JOIN. The most important function here is pd.merge().

python
# Two DataFrames to combine
employees = pd.DataFrame({
    "emp_id": [1, 2, 3],
    "name":   ["Alice", "Bob", "Carol"]
})

salaries = pd.DataFrame({
    "emp_id": [1, 2, 4],
    "salary": [72000, 95000, 61000]
})

# INNER JOIN — only rows matching in both
pd.merge(employees, salaries, on="emp_id", how="inner")

# LEFT JOIN — all employees, NaN if no salary found
pd.merge(employees, salaries, on="emp_id", how="left")

# RIGHT JOIN — all salaries, NaN if no employee found
pd.merge(employees, salaries, on="emp_id", how="right")

# OUTER JOIN — all rows from both, NaN for gaps
pd.merge(employees, salaries, on="emp_id", how="outer")

# Stack DataFrames vertically (same columns)
combined = pd.concat([df1, df2], ignore_index=True)

Pivot Tables

Pivot tables summarize data across two categories — like an Excel pivot table.

python
# Average salary by dept and year band
pivot = df.pivot_table(
    values="salary",          # What to aggregate
    index="dept",             # Rows
    columns="band",           # Columns
    aggfunc="mean",           # How to aggregate
    fill_value=0              # Replace NaN with 0
)
print(pivot)

# Simple cross-tab: count occurrences
pd.crosstab(df["dept"], df["band"])

Dates & Times

pandas has excellent support for time series data. Always convert date columns with pd.to_datetime() first.

python
# Parse a string column as dates
df["hire_date"] = pd.to_datetime(df["hire_date"])

# Extract date parts using .dt accessor
df["year"]    = df["hire_date"].dt.year
df["month"]   = df["hire_date"].dt.month
df["day"]     = df["hire_date"].dt.day
df["weekday"] = df["hire_date"].dt.day_name()  # e.g. "Monday"

# Calculate time differences
import pandas as pd
today = pd.Timestamp("today")
df["days_since_hire"] = (today - df["hire_date"]).dt.days

# Filter by date range
mask = (df["hire_date"] >= "2020-01-01") & (df["hire_date"] <= "2023-12-31")
df[mask]

# Resample time series data (e.g., monthly totals)
df.set_index("hire_date").resample("M")["salary"].sum()

Exporting Data

Save your cleaned or analyzed DataFrame back to a file:

python
# Save to CSV
df.to_csv("output.csv", index=False)   # index=False skips the row numbers

# Save to Excel
df.to_excel("output.xlsx", index=False, sheet_name="Results")

# Save multiple DataFrames to separate Excel sheets
with pd.ExcelWriter("multi_sheet.xlsx") as writer:
    df_sales.to_excel(writer, sheet_name="Sales", index=False)
    df_hr.to_excel(writer, sheet_name="HR", index=False)

# Save to JSON
df.to_json("output.json", orient="records")

# Print as a nicely formatted string
print(df.to_string())
