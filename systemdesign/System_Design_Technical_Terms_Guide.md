# Technical Terms Reference Guide

This document contains detailed descriptions of key technical terms identified in the uploaded System Design course material.

## System Design
The process of defining a software system's architecture, components, interfaces, and data flows to satisfy business and technical requirements.

## Distributed Systems
Systems whose components run on multiple networked computers and coordinate through communication protocols.

## Monolithic Architecture
A software architecture where all application functionality is packaged and deployed as a single unit.

## Microservices
An architectural approach that breaks an application into small, independently deployable services.

## Event-Driven Architecture
A design pattern where services communicate through events and asynchronous message exchanges.

## Scalability
The ability of a system to handle increasing workloads while maintaining acceptable performance.

## Reliability
The ability of a system to operate correctly and consistently over time.

## Fault Tolerance
The capability of a system to continue functioning even when some components fail.

## High Availability
A design objective focused on maximizing uptime and minimizing service interruptions.

## Load Balancing
The process of distributing traffic or workloads across multiple servers to improve performance and resilience.

## Caching
The practice of storing frequently accessed data closer to users or applications to reduce latency.

## DNS
The Domain Name System translates human-readable domain names into machine-readable IP addresses.

## IPv4
A 32-bit Internet Protocol addressing scheme supporting approximately 4.3 billion unique addresses.

## IPv6
A 128-bit Internet Protocol addressing scheme designed to provide a vastly larger address space.

## API Gateway
A centralized service that manages API traffic, routing, authentication, monitoring, and rate limiting.

## REST
An architectural style for APIs that uses HTTP methods and stateless communication.

## GraphQL
A query language and API runtime that allows clients to request exactly the data they need.

## gRPC
A high-performance remote procedure call framework that uses Protocol Buffers and HTTP/2.

## TCP
A reliable, connection-oriented transport protocol that guarantees ordered data delivery.

## UDP
A lightweight, connectionless transport protocol optimized for speed and low overhead.

## HTTP
The primary protocol used for transferring web content between clients and servers.

## HTTPS
A secure version of HTTP that encrypts data using TLS/SSL.

## WebSocket
A protocol that enables persistent, bidirectional communication between clients and servers.

## SQL
A language and database model used for managing structured relational data.

## NoSQL
A category of databases optimized for flexibility, scalability, and distributed operation.

## MongoDB
A document-oriented NoSQL database that stores information in JSON-like documents.

## Cassandra
A distributed NoSQL database designed for high availability and horizontal scalability.

## Database Sharding
A technique for splitting a database across multiple servers to improve performance and scalability.

## Indexing
A database optimization technique that speeds up query execution.

## Normalization
A database design process that reduces data redundancy and improves integrity.

## Denormalization
A technique that introduces controlled redundancy to improve read performance.

## Message Queue
A communication mechanism that enables asynchronous message exchange between systems.

## Kafka
A distributed event-streaming platform used for real-time data processing.

## RabbitMQ
A message broker that supports reliable message delivery and routing.

## Docker
A platform used to package applications and dependencies into portable containers.

## Kubernetes
A container orchestration platform for deploying, managing, and scaling containerized applications.

## Serverless Computing
A cloud execution model where infrastructure management is handled by the cloud provider.

## Cloud Computing
The on-demand delivery of computing resources and services over the internet.

## AWS
Amazon Web Services, a leading cloud computing platform offering infrastructure and managed services.

## Azure
Microsoft's cloud computing platform.

## Google Cloud Platform
Google's suite of cloud computing services and infrastructure.

## Authentication
The process of verifying the identity of a user or system.

## Authorization
The process of determining what actions an authenticated entity may perform.

## Encryption
The conversion of data into a secure format that can only be accessed with a key.

## Firewall
A security mechanism that controls network traffic based on predefined rules.

## DDoS Attack
A distributed denial-of-service attack that overwhelms a system with excessive traffic.

## CDN
A Content Delivery Network that serves content from geographically distributed servers.

## Proxy
An intermediary server that forwards requests between clients and destination systems.

## Forward Proxy
A proxy server that represents clients when accessing external resources.

## Reverse Proxy
A proxy server that represents backend services to incoming clients.

## CAP Theorem
A distributed systems principle stating that consistency, availability, and partition tolerance cannot all be fully guaranteed simultaneously.

## Consistency
The guarantee that all users see the same data at the same time.

## Availability
The guarantee that a system continues responding to requests even during failures.

## Client-Server Model
A computing model where clients request services and servers provide responses.
